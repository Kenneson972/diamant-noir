## Redirection admin vs dashboard proprio

- **Problème**: compte admin encore dirigé vers `/dashboard` (zone propriétaire) après login.
- **Correctifs**:
  - `postLoginDestination` dans `lib/auth/admin-access.ts`.
  - Login mot de passe, callback OAuth/magic link, flux OTP locataire alignés.
  - Middleware : admin sur `/dashboard/*` (hors `/dashboard/proprio`) → `/admin` ; `redirectWithSessionCookies` pour préserver les cookies Supabase.
- **Vérif**: `npm run build` OK.

## Middleware : espace client à tort pour les proprios

- **Cause**: `role === "client"` venait presque toujours des **JWT** (`user_metadata` absent), alors que `profiles.role` = `proprio` / `owner`.
- **Correctif**: `needsProfileForRbac` inclut `/espace-client` ; `isOwnerRole()` ; redirections basées sur profil + metadata.
