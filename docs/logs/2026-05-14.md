# Session 2026-05-14 — Admin : Liaison Propriétaire → Villa

## Contexte

Suite du brainstorming sur l'espace admin. Objectif : permettre à l'admin de gérer la relation propriétaire → villa, visualiser les infos du propriétaire et les réservations associées directement depuis la fiche villa.

## Actions réalisées

### 1. Liste des villas — Nom du propriétaire au lieu de l'ID

**Fichiers** : `app/(admin)/admin/villas/page.tsx`

- Ajout d'une fonction `getOwnersMap()` qui récupère les profils owner/proprio
- Remplacement de l'ID brut (UUID tronqué) par le nom complet du propriétaire
- Lien cliquable vers `/admin/membres/{ownerId}`

### 2. Formulaire ajout villa — Dropdown propriétaire

**Fichiers** : `components/dashboard/admin/AdminVillaForm.tsx`, `app/api/admin/owners/route.ts`, `app/api/dashboard/create-villa/route.ts`

- Création d'une API route `/api/admin/owners` qui liste les profils avec le rôle `owner` ou `proprio`
- Ajout d'un state `useState` pour charger les propriétaires au montage du formulaire
- Dropdown `<select>` permettant de choisir le propriétaire lors de la création de la villa
- Modification de l'API `create-villa` pour accepter un `owner_id` optionnel (fallback sur l'admin connecté)

### 3. Fiche villa détaillée — Bloc proprio + réservations récentes

**Fichiers** : `app/(admin)/admin/villas/[id]/page.tsx`

- Ajout d'un bloc **Propriétaire** dans la colonne principale :
  - Nom (lien vers fiche membre)
  - Email, téléphone
  - Statut Stripe Connect (connecté / en attente / non connecté)
- Ajout d'un bloc **Réservations récentes** (10 dernières) :
  - Nom et email du client
  - Montant, source, statut
  - Badge de statut coloré (confirmée / en attente / annulée)
- Requêtes parallélisées via `Promise.all` pour villa + owner + bookings
- Ajout de `force-dynamic` pour éviter le cache statique

## Correction post-session — Split paiement

**Problème signalé** : le split Stripe Connect ne respectait pas la FAQ Kayvila.

**Modèle correct** (FAQ thème « Commission & frais de ménage » + « Encaissement & reversements ») :
| Part | Bénéficiaire |
|------|--------------|
| 80 % des nuitées | Propriétaire |
| 20 % du séjour | Kayvila (commission) |
| 100 % frais de ménage | Kayvila |
| 100 % frais de service | Kayvila |

**Bug** : `ownerAmount = total - 20% séjour` → le proprio recevait aussi ménage + service.

**Fix** : `lib/stripe/connect.ts` + `app/api/booking/route.ts`
