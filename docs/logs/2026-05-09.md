## [20:00] Bug critique : villas absentes sur l'index (localhost)

- **Type**: `config | perf`
- **Summary**: Après un hard refresh, la section villas affichait "Aucune villa disponible" avec `TypeError: fetch failed`. Investigation :
  1. **Cause racine** : le sandbox réseau de Cursor bloque la résolution DNS de `*.supabase.co` → `getaddrinfo ENOTFOUND`. Le processus `next dev` est lancé depuis un terminal sandboxé.
  2. **`getSupabaseServer()`** échouait → `getaddrinfo ENOTFOUND` → catch externe → erreur silencieuse.
  3. **Modifications successives** :
     a. Ajout d'un fallback `createClient` basique (ne résolvait pas le problème DNS).
     b. Retrait de la condition `featuredVillas.length > 0` pour afficher l'état vide.
     c. Remplacement de `getSupabaseServer()` par `createClient()` direct.
     d. Remplacement du `fetch` patched Next.js par `https.get` Node.js natif.
     e. Relance du serveur avec `required_permissions: ["full_network"]` → **résolu**.
  4. **Modifications structurelles** : ajout de `export const dynamic = "force-dynamic"` pour éviter le cache Next.js ; `revalidate` retiré (incompatible avec `force-dynamic`).
- **Files**: [`app/page.tsx`, `next.config.mjs`]
- **Why**: Bug bloquant l'affichage des villas sur l'index en développement local.
- **Impact**: Les 3 villas les plus récentes s'affichent correctement sur la page d'accueil.
- **Verify**: `curl` vers localhost retourne `villa-hero` 9 occurrences ; confirmation visuelle navigateur.
