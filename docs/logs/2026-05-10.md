## Immersion 360° — Indicateur de progression par pilier

- **Type**: `ui`

Remplace le trait `h-px w-10` par 5 segments horizontaux pill (`h-[3px] flex-1 rounded-full`) : les segments jusqu'à l'index actif sont `bg-gold` (plein doré), les suivants `bg-gold/20` (ton beige transparent). ## Navbar — Correction troncature "À propos" sur md

- **Type**: `ui`
- **Summary**: Le lien "À propos" était visuellement tronqué sur les écrans md (768–1024px) à cause d'un dépassement de capacité dans la colonne droite du grid `[1fr_auto_1fr]`. En présence de 2 liens nav, icône téléphone, icône user et CTA texte plein, la colonne 1fr était saturée. Corrections :
  1. Icône téléphone : `hidden sm:flex` → `hidden lg:flex` (masquée jusqu'à 1024px).
  2. CTA texte/icône : expansion texte à partir de `lg:` au lieu de `md:` (`md:h-auto...` → `lg:h-auto...`) ; icône Sparkles masquée à partir de `lg:` au lieu de `md:`.
- **Files**: [`components/layout/Navbar.tsx`]
- **Why**: "À propos" s'affichait tronqué — le "À" était clipé hors du container flex sur les petits desktop.
- **Impact**: Tous les liens nav lisibles intégralement sur md ; plus d'espace dans la colonne droite.
- **Verify**: `npx tsc --noEmit` OK.

## Mobile — Défilement horizontal supprimé (`overflow-x: clip`)

- **Type**: `perf | ui`
- **Summary**: Le site défilait latéralement sur mobile malgré `overflow-x: hidden` sur `html, body`. Séparation des deux règles et passage de l'élément `html` à `overflow-x: clip`. Contrairement à `hidden`, `clip` ne crée pas de nouveau scroll container — les éléments `position: fixed` (navbar, drawer) ne sont pas affectés.
- **Files**: [`app/globals.css`]
- **Why**: "Il faut glisser vers la droite pour voir l'intégralité de la page" — overflow mobile non résolu.
- **Impact**: Suppression du défilement horizontal non intentionnel sur tous les écrans mobiles.
- **Verify**: `npx tsc --noEmit` OK.
