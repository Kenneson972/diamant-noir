# Vision — Le Nouvel Espace Client Kayvila

> Document de synthèse pour la direction — Mai 2026

---

## Aujourd'hui vs Demain

| | **Espace Client actuel** | **Nouvel Espace Client** |
|---|---|---|
| Le voyageur peut... | Consulter son séjour, voir un livret PDF, envoyer un message | **Agir** : demander un early check-in, signaler un problème, modifier ses dates, télécharger sa facture |
| L'équipe Kayvila peut... | Répondre aux messages manuellement | **Tout suivre en temps réel** : chaque demande devient une tâche avec statut, l'historique est tracé |
| Le Check-in... | Le voyageur appelle pour le digicode | Le voyageur reçoit le code **automatiquement 24h avant**, avec photos et plan d'accès |
| Le Check-out... | Aucune instruction standardisée | Checklist claire avec photos, rappel automatique la veille |
| Les documents... | Rien | Facture PDF automatique après le séjour, attestation sur demande |

**En bref** : on passe d'un espace "vitrine" à un espace "utile". Le voyageur n'appelle plus — il trouve tout dans son espace.

## Les 3 Phases

Nous construisons l'espace client en 3 phases. Chaque phase apporte de la valeur immédiate, sans attendre les suivantes.

### Phase 1 — Essentiel (maintenant)

Ce qu'on livre en premier, le minimum pour que l'espace soit vraiment utile.

| Fonctionnalité | Ce que ça change |
|---|---|
| **Check-in autonome** | Le voyageur reçoit le code du digicode 24h avant. Photos, plan, itinéraire depuis l'aéroport. Zéro appel. |
| **Profil voyageur complet** | Allergies, régime, occasion spéciale (anniversaire), heure d'arrivée, lit bébé. L'équipe sait tout avant l'arrivée. |
| **Règles & manuel de la villa** | WiFi, climatisation, jacuzzi, poubelles, barbecue. Photos pour chaque équipement. Le voyageur n'appelle pas pour savoir comment allumer le spa. |
| **Check-out instructions** | Checklist affichée la veille du départ. "Sortir les poubelles, fermer les volets, déposer les clés." Photos incluses. Demande de late check-out en 1 clic. |
| **Facture PDF automatique** | Téléchargeable après le séjour. Logo Kayvila, détails complets. Zéro manip manuelle. |
| **Système de demandes** | Le voyageur fait une demande (modif dates, problème, service). L'équipe Kayvila la voit dans son dashboard, clique Accepter ou Refuser. Le voyageur voit le statut en direct : "En attente" → "Accepté". Tout est tracé. |

### Phase 2 — Confort (ensuite)

Une fois la base solide, on enrichit l'expérience.

| Fonctionnalité | Ce que ça change |
|---|---|
| **Partager le séjour** | Le voyageur envoie un lien à son conjoint ou ses amis. Ils voient l'adresse, le WiFi, le check-in. Pas besoin de compte. |
| **Ajouter au calendrier** | 1 clic → le séjour apparaît dans Google Calendar / iPhone. |
| **Guide des alentours Kayvila** | Restaurants, plages, randonnées, marchés — recommandés par l'équipe. Photo, distance, note. |
| **Services ponctuels** | Ménage supplémentaire, changement de linge. En 1 clic. Prix affiché. L'équipe valide côté admin. |
| **Politique d'annulation** | Visible en clair. "Si vous annulez aujourd'hui, vous seriez remboursé de X €." Le voyageur comprend sans appeler. |

### Phase 3 — Fidélisation (plus tard)

L'espace devient un outil de rétention et de bouche-à-oreille.

| Fonctionnalité | Ce que ça change |
|---|---|
| **Avis post-séjour** | Note 1-5 + commentaire + photos. L'équipe valide avant publication. Les avis apparaissent sur la fiche villa. |
| **Centre de notifications** | Cloche avec badge. "Check-in dans 7 jours", "Votre demande est acceptée", "Message du concierge". Le voyageur ne rate rien. |
| **Re-réserver** | "Envie de revenir ? Réservez à nouveau la Villa Corail." Suggestions personnalisées. |
| **Favoris** | Le voyageur sauvegarde des villas pour plus tard. Disponibilités visibles en un coup d'œil. |
| **Parrainage** | "Recommandez Kayvila à un ami." Suivi des filleuls. Récompense à définir. |

## L'Interconnexion — Ce qui fait la force du système

Chaque action du voyageur est **connectée** au dashboard de l'équipe Kayvila.

```
Le voyageur demande un early check-in
                    ↓
   L'équipe Kayvila reçoit une notification
                    ↓
   Elle clique "Accepter" dans le dashboard admin
                    ↓
   Le voyageur voit "Accepté — vous pouvez arriver à 12h"
   + reçoit un email de confirmation
```

Rien ne se perd. Tout est tracé. L'équipe peut voir l'historique complet de chaque voyageur : ses demandes, ses séjours précédents, ses préférences.
